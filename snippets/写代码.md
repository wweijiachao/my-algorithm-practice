# sort和lambda

在 C++ 中，`sort` 函数就像是一把“瑞士军刀”，它是机试中出现频率最高、最好用的工具之一。

要掌握它，你只需要理解两个核心概念：**范围**和**规则**。

------

## 1. `sort` 函数的基本用法

`sort` 位于 `<algorithm>` 头文件中。最基本的用法是对数组或容器（如 `vector`, `string`）进行升序排序。

### 语法

```
sort(起始地址, 结束地址);
```

注意：它是**左闭右开**区间，即包含起始位置，但不包含结束位置。

### 示例

C++

```
string s = "dcba";
sort(s.begin(), s.end()); // 对 string 排序，结果为 "abcd"

int a[] = {4, 3, 2, 1};
sort(a, a + 4); // 对数组排序，结果为 {1, 2, 3, 4}
```

------

## 2. 核心：`cmp` (比较) 函数是什么？

默认情况下，`sort` 是**从小到大**排序。如果你想改变这个顺序（比如从大到小），或者根据复杂的规则排序，你就需要提供一个 `cmp` 函数。

`cmp` 函数的逻辑非常直观：**它定义了“前一个元素”在什么情况下应该排在“后一个元素”前面。**

### 逻辑模板

C++

```
bool cmp(类型 a, 类型 b) {
    if (满足这个条件) return true;  // a 排在 b 前面
    return false;                  // b 排在 a 前面
}
```

### 示例：实现降序（从大到小）

C++

```
bool cmp(int a, int b) {
    return a > b; // 如果 a 比 b 大，那么 a 应该在前面
}

// 使用方式：作为第三个参数传入
sort(v.begin(), v.end(), cmp);
```

------

## 3. 进阶：结构体排序（复试机试高频考点）

复试中最常考的不是排数字，而是排“学生信息”、“成绩单”等。

**题目要求**：输入学生姓名和成绩，按成绩**从高到低**排序；若成绩相同，按姓名**字典序升序**排序。

C++

```
struct Student {
    string name;
    int score;
};

bool cmp(Student a, Student b) {
    if (a.score != b.score) {
        return a.score > b.score; // 成绩不同，高分在前
    }
    return a.name < b.name;       // 成绩相同，名字字典序小的在前
}
```

------

## 4. 一个让你看起来更专业的写法：Lambda 表达式

在考场上，为了节省定义全局函数的时间，很多选手喜欢直接在 `sort` 里面写比较逻辑：

C++

```
sort(s.begin(), s.end(), [](char a, char b) {
    return tolower(a) < tolower(b); // 忽略大小写升序排序
});
```

------

## 💡 考前避坑小贴士

1. **不要在 `cmp` 里写等于号**：`cmp` 函数应定义“严格弱序”。即当 $a == b$ 时，必须返回 `false`。写成 `a >= b` 可能会导致某些编译器环境下程序崩溃。
2. **头文件别忘了**：一定要写 `#include <algorithm>`。
3. **效率**：`sort` 的复杂度是 $O(n \log n)$，在 1 秒内通常能处理 $10^5$ 到 $10^6$ 量级的数据。对于你刚才提到的“长度 200”的字符串，简直是瞬间完成。

# kruskal 优化 ky39

#### 1.性能优化：预分配内存 (`reserve`)

在 `calculateEdge` 中，你要往 `edges` 里塞 $N(N-1)/2$ 个元素。如果 $N$ 很大，`vector` 会多次触发自动扩容（申请新内存 -> 搬运数据 -> 释放旧内存），这很耗时。

**建议添加一行：**

C++

```
void calculateEdge() {
    // 预先分配内存，避免扩容开销
    edges.reserve(point_nums * (point_nums - 1) / 2); 
    
    for (int i = 0; i < point_nums; i++) {
        // ...
    }
}
```

#### 2. 逻辑优化：提前结束循环

目前的 `kruskal` 会遍历所有的边。实际上，一旦我们选够了 $N-1$ 条边，最小生成树就已经构成了，后面的边全是多余的。

**建议添加计数器：**

C++

```
void kruskal() {
    sort(edges.begin(), edges.end());
    int count = 0; // 记录已选边数
    for (auto& a : edges) {
        int i = a.i, j = a.j;
        if (find(i) != find(j)) {
            resultDistant += a.len;
            merge(i, j);
            count++;
            if (count == point_nums - 1) break; // 树已成，提前下班
        }
    }
}
```

#### 3. 构造函数的小细节

`Solution(int n)` 里，`unionSet` 的初始化循环：

C++

```
for (int i = 0; i < n; i++) unionSet[i] = i;
```

可以用 C++ 标准库的 `std::iota` (在 `<numeric>` 头文件) 替代，看起来更像“老手”：

# iota

在 C++ 中，`iota` 是一个非常精简但极其高效的工具函数，名字来源于希腊字母 **$\iota$ (iota)**。它定义在 `<numeric>` 头文件中，主要功能是：**用一组连续递增的数值填充容器**。

作为准研究生，在处理算法题（尤其是并查集初始化、离散化、或者是建立索引数组）时，它是替代手动 `for` 循环填充数据的首选。

------

### 1. 基本语法

`iota` 的函数签名如下：

C++

```
template< class ForwardIt, class T >
void iota( ForwardIt first, ForwardIt last, T value );
```

- **`first`, `last`**: 迭代器范围（左闭右开），定义了你要填充的区间。
- **`value`**: 起始值。第一个元素将被赋值为 `value`，第二个为 `value + 1`，以此类推。

------

### 2. 核心使用场景

#### A. 并查集初始化（你的代码中可以用的地方）

在并查集中，初始状态下每个节点的父节点都是它自己。

- **以前的做法**：使用 `for` 循环手动赋值。

- **现在的做法**：

  C++

  ```
  #include <numeric>
  vector<int> unionSet(n + 1);
  iota(unionSet.begin(), unionSet.end(), 0); // 填充 0, 1, 2, ..., n
  ```

#### B. 生成索引数组（用于自定义排序）

当你不想改变原数组的顺序，但想得到一个“按原数组值排序后的索引顺序”时，`iota` 非常有用。

C++

```
vector<int> data = {50, 10, 30, 20};
vector<int> idx(data.size());

iota(idx.begin(), idx.end(), 0); // 生成 {0, 1, 2, 3}

// 按 data 的值对索引 idx 进行排序
sort(idx.begin(), idx.end(), [&](int a, int b) {
    return data[a] < data[b];
});
// 结果 idx 变为 {1, 3, 2, 0}，对应原数组中 10, 20, 30, 50 的下标
```

------

### 3. 进阶特性与细节

#### 类型溢出风险

`iota` 内部使用的是 `++value`。如果你的起始值类型和容器类型不匹配，或者数值过大，可能会发生溢出。

C++

```
vector<long long> v(10);
iota(v.begin(), v.end(), 2147483647); // 如果起始值是 int，+1 就会溢出
// 建议明确指定类型：
iota(v.begin(), v.end(), 2147483647LL); 
```

#### 它不仅限于整数

只要对象支持 `operator++`，`iota` 就能工作。比如你可以用它填充连续的字符：

C++

```
vector<char> alphabet(26);
iota(alphabet.begin(), alphabet.end(), 'a'); // 填充 'a' 到 'z'
```

------

### 4. 为什么建议你习惯使用它？

1. **代码语义化**：看到 `iota` 别人立刻就知道你在做“连续初始化”，而 `for` 循环可能包含其他逻辑。
2. **减少 Bug**：手动写循环容易出现 `i <= n` 还是 `i < n` 的越界错误，`iota` 配合迭代器（`.begin()`, `.end()`）是绝对安全的。
3. **编译器优化**：作为标准库函数，编译器通常会对 `iota` 进行高度优化（如向量化指令），执行效率通常优于或等于手写循环。

### 💡 师兄点评

在你的并查集实现中，构造函数里的 `for (int i = 0; i <= vNums; i++) unionSet[i] = i;` 建议直接换成 `iota(unionSet.begin(), unionSet.end(), 0);`。这不仅显得你对标准库很熟悉，也符合现代 C++ (Modern C++) 的编程风格。

### `<numeric>` 头文件四大神技

除了刚才提到的 `iota`，这四个函数在处理数组、前缀和、以及差分题目时非常强大：

#### 1. `accumulate` (累加/求和)

不用再手写 `sum += a[i]`。

- **用法**：`accumulate(begin, end, init_value)`
- **技巧**：第三个参数 `init_value` 的类型决定了结果类型。
  - `accumulate(v.begin(), v.end(), 0)` -> 返回 `int`。
  - `accumulate(v.begin(), v.end(), 0LL)` -> 返回 `long long`（**避开溢出大坑**）。

#### 2. `partial_sum` (生成前缀和)

前缀和是算法题的常客（比如区间求和）。

- **用法**：`partial_sum(src_begin, src_end, dest_begin)`
- **效果**：若原数组是 `{1, 2, 3}`，生成后变成 `{1, 3, 6}`。

#### 3. `adjacent_difference` (生成差分/相邻差)

前缀和的逆运算，常用于区间加减操作。

- **用法**：`adjacent_difference(src_begin, src_end, dest_begin)`
- **效果**：若原数组是 `{1, 3, 6}`，生成后变成 `{1, 2, 3}`。

#### 4. `gcd` & `lcm` (最大公约数 & 最小公倍数)

*C++17 之后直接内置在 `<numeric>` 中，不需要再自己写辗转相除法了！*

- **用法**：`gcd(a, b)` 或 `lcm(a, b)`。

# `emplace_back` 和 `push_back`

在 90% 的情况下 `emplace_back` 的性能 $\ge$ `push_back`，但在某些特定场景下，它们**性能完全一样**，甚至 `emplace_back` 会带来**代码安全性问题**

### 1. 什么时候 `emplace_back` 完胜？（构造参数时）

这是 `emplace_back` 诞生的初衷：**利用参数原地构造**。

C++

```
struct Student {
    Student(int id, int score) { ... }
};

vector<Student> v;

// ❌ push_back:
// 1. 构造临时对象 Student(1, 90)
// 2. 移动构造到 vector 内存中
// 3. 析构临时对象
v.push_back(Student(1, 90)); 

// ✅ emplace_back:
// 1. 直接在 vector 内存中调用 Student(1, 90)
v.emplace_back(1, 90); 
```

**结论**：只要你传的是**构造函数的参数列表**（而不是对象本身），`emplace_back` 完胜。它省去了移动和析构的开销。

---

### 2. 什么时候两者性能完全一样？（传入已有对象时）

如果你已经有一个现成的对象（无论是左值还是右值），两者的底层行为几乎没有区别。

#### 情况 A：传入左值 (L-value)

C++

```
Student s(1, 90);

// 此时必须拷贝 s，无法避免
v.push_back(s);    // 调用拷贝构造函数
v.emplace_back(s); // 调用拷贝构造函数
```

- **解析**：`emplace_back` 内部只是把 `s` 转发给了构造函数，结果还是拷贝。性能 **100% 相同**。
    

#### 情况 B：传入右值 (R-value/xvalue)

C++

```
Student s(1, 90);

// 此时通过 std::move 触发移动
v.push_back(std::move(s));    // 调用移动构造函数
v.emplace_back(std::move(s)); // 调用移动构造函数
```

- **解析**：同上，两者都执行移动操作。性能 **100% 相同**。
    

---

### 3. 什么时候 `emplace_back` 是“危险”的？（隐式转换陷阱）

这是面试中最高级的考点。`emplace_back` 过于强大，有时会**绕过 `explicit` 关键字的检查**，导致你写出原本应该报错的 Bug 代码。

假设你的 `vector` 是存 `vector<int>` 的（二维数组）：

C++

```
vector<vector<int>> matrix;

// 你的意图：想往里塞一个数字 10？不可能吧，matrix 的元素是 vector
// matrix.push_back(10);  
// ❌ 编译报错！因为 vector(int size) 构造函数是 explicit 的，禁止隐式将 int 转为 vector。
// 这保护了你，告诉你类型不对。

// matrix.emplace_back(10); 
// ✅ 编译通过！运行正常！但结果是你得到了一行长度为 10 的空 vector。
```

**原因**：

- `push_back` 接受的是 `T` 类型，如果不匹配，编译器会尝试隐式转换。如果构造函数是 `explicit`，转换失败，报错。
    
- `emplace_back` 是个模板函数，它接受 `Args...`，然后直接去调用构造函数 `T(Args...)`。它不进行隐式转换检查，直接霸王硬上弓。
    

**结论**：在这种情况下，`emplace_back` 虽然能跑，但它破坏了**类型安全**。如果你写错了代码，`push_back` 会拦住你，而 `emplace_back` 会默默地制造一个诡异的 Bug。

---

### 4. 什么时候 `emplace_back` 会导致内存泄漏？（极罕见但存在）

这涉及到智能指针和异常安全。

C++

```
// 假设 process 可能会在 vector 扩容时抛出异常
vector<shared_ptr<int>> v;

// 写法 1：安全
v.push_back(make_shared<int>(100));

// 写法 2：不安全（在极老的编译器或特定优化下）
v.emplace_back(new int(100)); 
```

**解析**：

如果在 `emplace_back` 内部，`new int(100)` 执行了，但在将其构造为 `shared_ptr` 存入 `vector` 之前，`vector` 因为扩容失败抛出了异常……

- 那个裸指针 `new int(100)` 就没人管了 $\rightarrow$ **内存泄漏**。
    
- 虽然后来的 C++ 标准修复了参数求值顺序，但这依然被视为一种不好的代码风格（Bad Smell）。
    

---

### 💡 最佳实践指南

既然你追求代码的**高性能**与**高可靠性**，建议遵循以下原则：

1. **首选 `emplace_back`**：当你需要**用参数列表**创建一个新对象时。

    - _例：_ `list.emplace_back(1, 90, 80, 70);` （你的代码现在的样子，完美）

2. **用 `push_back`**：当你已经有一个**现成的对象**（不管是 `s` 还是 `std::move(s)`）时。

    - _理由：_ 打字少，且编译器给的错误信息更清晰。

3. **用 `push_back`**：当你希望编译器帮你检查**类型安全**，防止意外的隐式转换时。

    - _例：_ `vector<vector<int>>` 这种嵌套容器。

# `std::unique`数组去重

`std::unique` 可以去除所有**相邻的**重复元素，并返回去除后数组的结束位置（新最后一个元素的位置+1，也即是迭代器`.end()`）。
配合`std::sort()`、`vector.erase()`函数可以实现数组排序和去重。

```cpp
std::sort(arr.begin(), arr.end());
auto new_end = std::unique(arr.begin(), arr.end());
arr.erase(new_end, end());
```

# 质数

```cpp

// 埃氏筛，找出从3到 n的所有质数,O(NloglogN)，非常接近O(N)
vector<bool> era_sieve(int n) {
    vector<bool> is_prime(n + 1, true);
    is_prime[0] = is_prime[1] = false;

    for (int i = 2; i <= n / i; i++) {
        if (is_prime[i]) {
            for (int j = i * i; j <= n; j += i) {
                is_prime[j] = false;
            }
        }
    }
    return is_prime;
}


// 欧拉筛：时间复杂度：严格的 O(N)
void euler_sieve(int n) {
    vector<bool> is_prime(n + 1, true);
    vector<int> primes; // 用来存已经找到的质数
    is_prime[0] = is_prime[1] = false;

    for (int i = 2; i <= n; i++) {
        if (is_prime[i]) {
            primes.push_back(i); // 如果没被筛掉，它就是质数
        }
        // 遍历已经找到的质数
        for (int j = 0; j < primes.size() && i * primes[j] <= n; j++) {
            is_prime[i * primes[j]] = false; // 质数 * 任意数 = 合数，筛掉
            
            // 【核心灵魂语句】
            // 如果 i 包含了当前的质因数 primes[j]，立刻 break！
            // 保证每个合数只被它的最小质因数筛掉
            if (i % primes[j] == 0) {
                break;
            }
        }
    }
}
```

# 欧几里得算法（辗转相除法）

------

## 一、 最大公约数 (GCD) —— 绝对的灵魂模板

**核心数学原理：** 两个数 $a$ 和 $b$ 的最大公约数，等于 $b$ 和 $a \pmod b$ 的最大公约数。

当其中一个数变成 0 时，另一个数就是我们要找的最大公约数。

**C++ 模板代码：**

C++

```cpp
// 时间复杂度：O(log(min(a, b)))，极其快
int gcd(int a, int b) {
    // 如果 b 是 0，返回 a；否则递归调用 gcd(b, a % b)
    return b == 0 ? a : gcd(b, a % b);
}
```

**💡 机试必知细节：**

1. **不用管大小顺序：** 你可能会问：“如果传进来的参数 $a < b$ 怎么办？” 完全不用担心！如果 $a = 15, b = 20$，$15 \pmod{20} = 15$。下一次递归时，参数自动变成了 `gcd(20, 15)`，它自己会把大的数换到前面去。

2. **C++17 的“官方外挂”：** 如果你报考学校的机试系统比较新（支持 C++17），你可以直接包含头文件 `#include <numeric>`，然后直接调用 `std::gcd(a, b)`，连这行代码都不用手写！但为了防备老旧的机考系统，手写这行三目运算符是最稳的。
  
------

## 二、 最小公倍数 (LCM) —— 连环套娃

考了 GCD，大概率就会顺带考 LCM。

这俩兄弟有一个非常死板的数学绑定关系：**两个数的乘积，等于它们的最大公约数乘上最小公倍数。** 即 $a \times b = gcd(a, b) \times lcm(a, b)$。

所以，求 LCM 的本质就是求 GCD。

**C++ 模板代码：**

C++

```cpp
int lcm(int a, int b) {
    // ⚠️ 极其致命的考点：必须先除后乘！
    return a / gcd(a, b) * b; 
}
```

**☠️ 避坑指南（无数人在这里丢分）：**

很多同学会根据公式，本能地写成 `return (a * b) / gcd(a, b);`。

**绝对不要这么写！** 如果 $a$ 和 $b$ 都很大（比如 10 万级别），`a * b` 在做除法之前就会瞬间超出 `int` 的最大范围（2147483647），导致整型溢出，变成负数，直接爆零。

**先让 $a$ 除以 GCD 把数字降下来，再去乘 $b$，这是算法竞赛圈的基本常识。**
